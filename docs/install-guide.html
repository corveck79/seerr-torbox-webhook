1Check requirementsVereisten controleren

Docker and Docker Compose are already present via the Proxmox LXC template. Verify:

Docker en Docker Compose zijn al aanwezig via de Proxmox LXC template. Even checken:

docker --version
docker compose version

Install Container Manager (Synology) or Container Station (QNAP) from the package center. Docker Compose is included. Then verify via SSH:

Installeer Container Manager (Synology) of Container Station (QNAP) vanuit het pakketcentrum. Docker Compose is inbegrepen. Controleer daarna via SSH:

docker --version
docker compose version
ℹ You need at least Docker Compose v2.x (docker compose, not docker-compose). Je hebt minimaal Docker Compose v2.x nodig (docker compose, niet docker-compose).

Beide bestanden horen in /opt/Myc3l1um/:

scp docker-compose.yml .env root@<your-ip>:/opt/Myc3l1um/
⚠ Make sure both files are placed exactly in /opt/Myc3l1um/. Docker Compose looks for .env next to docker-compose.yml. Zorg dat beide bestanden exact in /opt/Myc3l1um/ staan. Docker Compose zoekt .env naast docker-compose.yml.

Both files belong in /volume1/docker/mycelium/. Upload them via File Station or copy them over an SMB share:

Beide bestanden horen in /volume1/docker/mycelium/. Upload ze via File Station of kopieer ze via een SMB-share:

10Understanding streams and 404 errorsStreams en 404-fouten begrijpen

Mycelium uses catbox mode: the .strm files contain a proxy URL (/stream/<token>). Torrents are NOT added to TorBox during request processing. Instead, the torrent is added on-demand when you press play in Jellyfin. Mycelium then finds a cached release, adds it to TorBox, waits for it to be ready, and redirects to the CDN stream.

Mycelium gebruikt catbox mode: de .strm bestanden bevatten een proxy-URL (/stream/<token>). Torrents worden NIET aan TorBox toegevoegd tijdens het verwerken van aanvragen. In plaats daarvan wordt de torrent on-demand toegevoegd wanneer je op play drukt in Jellyfin. Mycelium zoekt dan een gecachede release, voegt deze toe aan TorBox, wacht tot deze klaar is, en stuurt door naar de CDN-stream.

ℹ A 404 on a stream means the token is unknown or the item has not been registered in Mycelium yet. This can happen if the .strm file was created by torbox-mc-strm instead of Mycelium, or if the database was reset. Items registered in Mycelium will work on first play, as the torrent is fetched on demand. Een 404 op een stream betekent dat het token onbekend is of dat het item nog niet in Mycelium is geregistreerd. Dit kan gebeuren als het .strm bestand is aangemaakt door torbox-mc-strm in plaats van Mycelium, of als de database is gereset. Items die in Mycelium zijn geregistreerd werken bij de eerste keer afspelen, omdat de torrent on-demand wordt opgehaald.

After inactivity (CATBOX_IDLE_MINUTES=60), the torrent is removed from TorBox to save space. The next time you play, it is re-added automatically. This cycle is normal and by design.

Na inactiviteit (CATBOX_IDLE_MINUTES=60) wordt de torrent verwijderd uit TorBox om ruimte te besparen. De volgende keer dat je afspeelt, wordt deze automatisch opnieuw toegevoegd. Dit is normaal en zo ontworpen.
